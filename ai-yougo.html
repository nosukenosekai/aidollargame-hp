GLOSSARY

AI用語集
むずかしい言葉を、やさしく。

AIの記事やニュースを読むと、横文字とカタカナばかりで疲れますよね。ここでは、AI導入の現場で実際によく出てくる言葉だけを集めて、専門知識がなくても分かるようにかみくだいて説明しています。気になった言葉から、つまみ食いでどうぞ。

基本のキ 使うときに出てくる言葉 AIエージェント・導入まわり これから耳にする言葉
01

基本のキ

まずはこの5つ。ニュースに出てくるAIは、だいたいこの言葉でできています。

生成AI
Generative AI

文章・画像・音声・プログラムなどを、ゼロから新しく“生成”するAIの総称です。ChatGPTやClaudeのように「お願いすると作ってくれる」タイプがこれにあたります。

LLM（大規模言語モデル）
Large Language Model

膨大な量の文章を学習して、人間のような自然な文章を理解・生成できるようにしたAIの中核技術です。ChatGPT・Claude・Geminiなどの“正体”がこれです。

機械学習
Machine Learning

大量のデータから、ルールやパターンをAI自身が自動で学ぶ仕組みです。人間が一つひとつルールを書かなくても、データさえあれば賢くなっていきます。

ディープラーニング（深層学習）
Deep Learning

機械学習の一種で、人間の脳の神経回路をまねた何層もの構造で、複雑なパターンを学びます。今のAIブームを支えている土台の技術です。

自然言語処理（NLP）
Natural Language Processing

人間が使う言葉を、コンピュータが扱えるようにする技術分野です。翻訳・要約・チャット対応など、「言葉を相手にするAI」の基盤になっています。

02

使うときに出てくる言葉

実際にAIを触り始めると、料金や使い勝手の話でこのあたりが出てきます。

プロンプト
Prompt

AIへの指示文、つまり「お願いの言葉」です。同じAIでも、頼み方しだいで答えの質が大きく変わります。AIを使いこなす=プロンプトが上手、と言ってもいいくらいです。

トークン
Token

AIが文章を処理するときの最小単位です。料金や処理量は、このトークン数で数えます。日本語だとおおよそ1文字＝1〜2トークンが目安です。

コンテキストウィンドウ
Context Window

AIが一度に覚えていられる文章量の上限です。ここが大きいほど、長い資料や会話履歴をまるごと渡して、それを踏まえた回答をさせられます。

ハルシネーション
Hallucination

AIが、もっともらしい“ウソ”や誤りを、自信満々に答えてしまう現象です。AIの回答をそのまま鵜呑みにせず、事実確認が必要な理由がこれです。

マルチモーダル
Multimodal

文章だけでなく、画像・音声・動画なども一緒に理解したり生成したりできること。写真を見せて「これは何？」と聞けるAIは、マルチモーダルです。

推論
Inference

学習を終えたAIが、実際に答えを出す処理のことです。ざっくり言えば「AIを動かす＝推論させる」。最近は、じっくり考えてから答える“推論モデル”も増えています。

03

AIエージェント・導入まわり

「会社にAIを入れる」話になると登場する言葉です。ここが分かると検討がぐっと楽になります。

AIエージェント
AI Agent

質問に答えるだけでなく、自分で段取りを考えて作業を実行し、“やってくれる”AIです。問い合わせ対応・予約管理・調査などを代行します。 現場での意味：「答えるAI」から「動くAI」へ。当社のAIチャットエージェントもこのタイプです。

チャットボット
Chatbot

あらかじめ決めたシナリオやFAQに沿って会話する仕組みです。便利ですが、想定外の質問には弱い。AIエージェントより自律性は低めです。 ちがい：詳しくはAIエージェントとチャットボットの違いで解説しています。

RAG（検索拡張生成）
Retrieval-Augmented Generation

AIに社内マニュアルや自社データを参照させて、自分の会社の正確な情報で答えさせる仕組みです。ハルシネーション（誤答）を抑える、導入時の定番手法です。

ファインチューニング
Fine-tuning

既存のAIに自社のデータを追加で学習させ、専門分野や自社らしい口調に“仕立て直す”ことです。RAGと並んで、自社仕様にするときの選択肢になります。

埋め込み／ベクトル
Embedding

文章の“意味”を数値の列に変換したものです。意味の近い文書どうしを探せるようになり、RAGで「関係する資料」を見つけ出す土台になります。

API
Application Programming Interface

ソフトウェア同士をつなぐ「窓口」のことです。自社の予約システムや顧客管理にAIを組み込むときは、このAPI経由でつなぎます。

MCP
Model Context Protocol

AIを、外部のツールやデータに安全につなぐための共通規格です。AIエージェントが実際の業務をこなすための“配管”にあたり、近年急速に広まっています。

ガードレール
Guardrails

AIが不適切・危険・的外れな回答をしないように設ける、制御やルールのことです。会社で使うAIには、この“はみ出さない仕組み”が欠かせません。

04

これから耳にする言葉

今すぐ必要ではないけれど、ニュースで見かけたときに「あれね」と分かると安心な言葉です。

基盤モデル（ファウンデーションモデル）
Foundation Model

大量のデータで学習し、さまざまな用途に応用できる“土台”となる大型のAIモデルです。各社のLLMは、この基盤モデルの代表例です。

オープンウェイト／オープンソースAI
Open-weight / Open-source AI

モデルの中身（重み）が公開されていて、自社の環境でも動かせるAIです。社外にデータを出したくない用途で選ばれることがあります。

AGI（汎用人工知能）
Artificial General Intelligence

人間と同じように、幅広い知的作業を何でもこなせるAI——という、まだ実現していない“目標段階”を指す言葉です。ニュースで未来の話として登場します。

量子化
Quantization

AIモデルを軽く圧縮して、より安く・速く動かせるようにする技術です。性能を保ちつつコストを下げたいときに使われます。

蒸留
Distillation

大きくて賢いモデルの“知恵”を、小さなモデルに移し替えて軽量化する技術です。スマホや手元の端末でも動く小型AIを作るときに使われます。
