# 承継OS ── 引き継ぎ資料

これは、株式会社aidollargame の新規事業「承継OS」を、投資家向けに詰めた企画の一式です。
仮想役員会CLI（`board-cli/`）で5人に何度も議論させ、外部の目でのレビューも重ねて作りました。
この1枚を読めば、次の人が中身と経緯を引き継げるようにしてあります。

## この事業は何か（1行）

後継者がいなくて畳まれる黒字の会社を、**社長がいなくても回る（＝売れる・継げる）状態**に変えて、次の人へ渡す。
売るのは「準備」ではなく「**売れる・継げる会社にすること**」。

- 会社 = aidollargame（小さいAIネイティブ。本業はAI事務代行）
- 承継版 = 月30万×3〜6か月で現場に深く入る。そこで集まる「剥がした後どうなったか」のデータが堀。
- 目的 = **シード調達**（1人社長＋AIでは現場が回らない → チームを作るために調達する）。

## いま決まっていること（企画の骨）

- **最初の5社**：業種は決め打ちしない。「最初の客の条件」で選ぶ（黒字・売る気がある・社長依存が事務/段取り・データが取れる・払える）。導線は既存のAI顧客から派生が最速。
- **ファネル**：面談30 → 条件合致10 → 有償診断5 → 月額導入3 → 承継成立1 → 90日継続1。
- **収益**：診断50万／月額30万（3〜6か月）／成功報酬 譲渡額3〜5%／渡した後は見守り料・仕組み利用料が中心（少数持分は条件が揃った案件のみ）。
- **測り方**：「社長依存指数」（権限移譲率・業務標準化率・顧客直結度・財務即時性・非属人意思決定率）。
- **銀行**：置き換えでなく「補助の材料」として地銀1行で安く試す。利益相反の手当て（スコアを出す案件は月額のみ等）が生命線。
- **堀**：「継げる状態」と「承継後の成果」の相関を、実際に剥がして継がせた結果データで継続的に高められること。実務に入る当社しか持てない。
- **資金**：シード1.5〜3億円 → シリーズA 10億円規模。会社を買う資金は都度、銀行・公的融資で。
- **目標**：3年で再現性を証明し初期拡大（売上3〜5億円）、5年は金融接続が通った場合のアップサイド（20〜30億円規模）。時価総額目標は対外資料に出さない。

## 議論に使う「仮想役員会」の5人（← ここ重要）

議論はこの**決まった5人**で回す。**勝手に別のメンバー（日本人の役員など）を作らないこと。**
定義の元データは `board-cli/boards/default.json`。中身は以下の通り。

| 名前（ラベル） | 役割（レンズ） | 何を最優先するか |
|---|---|---|
| イーロン・マスク | 攻めの破壊的革新者 | 第一原理で常識を疑い、桁違いの成長と市場獲得 |
| ジェフ・ベゾス | 長期の戦略家（顧客執着） | 10年後も変わらない顧客価値・長期で勝てるか |
| チャーリー・マンガー | 辛口の懐疑派（悪魔の代弁者） | 前提の甘さ・破滅リスクを逆算で暴く（粗探し担当） |
| ウォーレン・バフェット | 本物を見抜く長期投資家 | 理解できる商売か・堀があり長く稼げるか |
| ピーター・ティール | 逆張りの独占主義者 | 競争を避け小さな市場を独占・他人が気づかない秘密 |

**固定ルール（毎回守る）:**
- 名前はあくまで「思考のレンズ」であって本人ではない。**「本人が実際にこう言った」とは書かない。**
- 数字や固有の事実には `[要裏取り]` を付ける。
- 難しいカタカナ用語・専門用語は使わない（普通の日本語で書く）。
- **毎回、5人それぞれの意見（議事録）を先に出してから**、議長が結論を1つに絞る。
- Claude自身の意見は要らない。5人に議論させた結果だけを持ってくる。

**実際にCLIで回す場合:** `board-cli/README.md` 参照。Gemini APIキーを環境変数 `GEMINI_API_KEY` に入れて `node board-cli/index.js` で動く。
キーが無い環境では、上の5人の設定になりきって議論をシミュレートしてもよい（ただしメンバーとルールは上のまま）。
