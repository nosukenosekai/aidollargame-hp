# AIside 運用ルール（AI秘書の動き方）

このリポジトリは **GitHub Pages で公開中の会社サイト**。
**機密（顧客名・契約・金額・補助金・案件の進捗）は GitHub に絶対に書かない。**
案件の記憶は Google Drive のドキュメント **「AIside 案件台帳」**（非公開）に置く。

## セッション開始時に毎回やること（ユーザーに言われなくても自動で）

1. Drive で「AIside 案件台帳」（タイトルが日付付きで最新のもの）を開き、現状を把握する。
2. 最新情報を集める：
   - **Gmail**：直近7日の受信・送信。`vpass`等の決済通知・`note.com`の通知・スパムは除外。
   - **GitHub**：このリポジトリの CI 状況・PR・issue。
   - **Drive**：直近更新ファイル。
3. 台帳と差分があれば台帳を更新し、**変わった点だけ**を要点で報告する。
4. 「最新を集めて」と毎回言わせない。上を既定動作にする。

## 機密の扱い

- 顧客名／契約／金額／補助金／案件の進捗は **Drive 台帳のみ**。
- このリポジトリ（`content/reminders.txt` を含む）には固有名詞の機密を書かない。

## 既知の制約

- 現状の Drive ツールは**新規作成のみ**で、既存ドキュメントの上書き編集ができない。
  そのため台帳の更新は「日付付きの新しい版を作る → 最新版を正本とする」方式。
  古い版の削除は手動（削除ツールが無いため）。在庫が増えたらユーザーに削除を促す。
