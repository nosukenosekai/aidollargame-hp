AI用語集｜ぜんぶ「たとえ話」でやさしく

「生成AI」「API」「LLM」——AIの言葉って、横文字ばかりで身構えますよね。でも、どれも一回たとえ話で分かれば、びっくりするほど簡単です。このページでは、AIの専門用語を専門用語を使わず、全部たとえ話で解説します。お子さんが読んでも分かるくらい、やさしくいきます。

生成AIせいせいエーアイ / Generative AI

ひとことで言うと：お願いすると、文章でも絵でも“作ってくれる”AI。

たとえ話：すごく物知りで、絵も上手な友だちを想像してください。「読書感想文こんな感じで書いて」「猫の絵かいて」と頼むと、サッと作ってくれる。あれの正体が生成AIです。ChatGPTもこの仲間。

LLMエルエルエム / 大規模言語モデル

ひとことで言うと：生成AIの“頭の中身”。言葉をあつかう脳みそ。

たとえ話：あの物知りな友だちが何でも知ってるのは、世界中の本や記事をとんでもない量、読んできたから。その「読んできた知識のかたまり」がLLMです。脳みそだと思ってください。

プロンプトPrompt

ひとことで言うと：AIへの“お願いの仕方”。

たとえ話：レストランの注文と同じ。「何かおいしいもの」より「辛くないチキンカレーを大盛りで」と言ったほうが欲しいものが出てくる。AIもお願いがくわしいほど、いい答えを返します。

トークンToken

ひとことで言うと：AIが文章を読むときの“ひとかたまり”の単位。

たとえ話：回転寿司で食べた量を「お皿の枚数」で数えますよね。AIも文章を小さなかたまりに分け、その数（トークン）で「どれだけ読んだ・書いたか」を数えます。長い文章ほどお皿が増えるイメージ。

APIエーピーアイ

ひとことで言うと：アプリとアプリをつなぐ“受け渡しの窓口”。

たとえ話：お店の「注文カウンター」を想像してください。厨房の中（複雑な仕組み）を知らなくても、カウンターで「これください」と言えば商品が出てくる。APIはこのカウンター役。LINEとAIをつなぐときも、この窓口を通して「お客さん来たよ」「こう返事して」とやりとりします。

AIエージェントAI Agent

ひとことで言うと：答えるだけじゃなく、“実際に動いてくれる”AI。

たとえ話：物知りな友だち（生成AI）は聞けば教えてくれる。でもエージェントはお屋敷の執事さん。「来週の出張、手配しといて」と言えば、調べて、予約して、カレンダーに入れて、報告までしてくれる。“考える”だけでなく“手を動かす”のが特徴です。

チャットボットChatbot

ひとことで言うと：決まった質問に決まった答えを返す自動応答。

たとえ話：チャットボットは自動販売機（決まったボタン＝決まった答え）。AIエージェントはホテルのコンシェルジュ（ふわっとした相談にも考えて応え、予約まで取る）。同じ「自動対応」でも賢さがまるで違います。

ハルシネーションHallucination

ひとことで言うと：AIが知らないことを、“それっぽく作って言っちゃう”こと。

たとえ話：「知らない」と言えばいいのに、見栄をはってそれっぽいウソを自信満々に答えちゃう友だち、いますよね。AIもたまにこれをやります。だから大事な情報は人間が最後に確認するのが鉄則です。

機械学習きかいがくしゅう / Machine Learning

ひとことで言うと：たくさんの例から、AIが“コツをつかむ”こと。

たとえ話：自転車の練習と同じ。最初は転ぶけど、何度もやるうちに体がコツを覚える。AIも山ほどの例を見ながら「こういうときはこう」を少しずつ身につけます。

ディープラーニング深層学習 / Deep Learning

ひとことで言うと：機械学習の中でも、特に賢いやり方。

たとえ話：1人で考えるより、何人もが順番に考えを深めていくと難しい問題も解ける。AIの中で考える層を何段も重ねて、複雑なことまで学べるようにしたのがディープラーニングです。

アルゴリズムAlgorithm

ひとことで言うと：ものごとを解くための“手順・レシピ”。

たとえ話：カレーの作り方の手順書と同じ。「材料を切る→炒める→煮る」のように、決まった順番で進めれば答えにたどり着く手順がアルゴリズムです。

モデルModel

ひとことで言うと：学習を終えたAIの“本体”。

たとえ話：たくさん練習して一人前になった職人さん。勉強（学習）を終えて、できあがった脳がモデルです。「このモデルは賢い」は「この職人は腕がいい」と同じ意味。

ファインチューニングFine-tuning

ひとことで言うと：できあがったAIに、追加で“専門の勉強”をさせること。

たとえ話：優秀な新人を採用したあと、自社のやり方を研修で教え込むイメージ。一般的に賢いAIに、自分の会社や業界の知識を追加で覚えさせるのがファインチューニングです。

RAGラグ / 検索拡張生成

ひとことで言うと：AIが答える前に“資料を調べてから”答える仕組み。

たとえ話：記憶だけで答えると間違えることもあるけど、手元の資料を見ながら答えれば正確。RAGは「カンニングOKのテスト」のようなもので、自社マニュアルを参照して正しく答えさせるのに使います。

マルチモーダルMultimodal

ひとことで言うと：文字だけでなく、画像・音声・動画もまとめて扱えるAI。

たとえ話：文字しか読めない状態から、目と耳も使えるようになったイメージ。写真を見せて「これ何？」と聞いたり、音声で話しかけたりできるAIがマルチモーダルです。

自然言語処理NLP / Natural Language Processing

ひとことで言うと：人間のふつうの言葉を、AIが理解・処理する技術。

たとえ話：外国語を日本語にしてくれる通訳さんのような役割。人間の言葉（自然言語）を、コンピュータが分かる形に“通訳”して、また人間の言葉で返してくれる技術です。

AIって、言葉がむずかしいだけ。中身はシンプルです。
