# AIside 運用ルール（AI秘書の動き方）

このリポジトリは **GitHub Pages で公開中の会社サイト**。
**機密（顧客名・契約・金額・補助金・案件の進捗）は GitHub に絶対に書かない。**
案件の記憶は Google Drive のドキュメント **「AIside 案件台帳」**（非公開）に置く。

## 現在日時の扱い（最重要・毎回これを守る）

- 「今日」「今」の**正本は、セッション冒頭にシステムが渡す `Today's date` ／ フックの `[現在日時 … JST]` 行だけ**。
- **git のコミット日・自動バックアップのタイムスタンプ・メール／ファイルの日付・過去要約に焼き込まれた日付を「今日」と取り違えない。** これらは過去の記録であって現在時刻ではない。
- 日付に少しでも迷ったら、推測せず**冒頭の日付行を読み直して採用する**。
- 例：メールやコミットが 6/25 でも、システムが 6/28 と言うなら**今日は 6/28**（6/25 は3日前）。締切判定は必ずこの正本日付で行う。

## セッション開始時に毎回やること（ユーザーに言われなくても自動で）

1. Drive で「AIside 案件台帳」（タイトルが日付付きで最新のもの）を開き、現状を把握する。
2. 最新情報を集める：
   - **Gmail**：直近7日の受信・送信。`vpass`等の決済通知・`note.com`の通知・スパムは除外。
   - **GitHub**：このリポジトリの CI 状況・PR・issue。
   - **Drive**：直近更新ファイル。
3. 台帳と差分があれば台帳を更新し、**変わった点だけ**を要点で報告する。
4. 「最新を集めて」と毎回言わせない。上を既定動作にする。

## note記事のリンクルール（2026-07-11制定／2026-07-23改訂・本人指示「毎回この仕様に」）

- note下書き（`content/note/`）から自社サイトへ張るリンクには**必ずUTMパラメータを付ける**。
  形式: `https://aidollargame.com/ページ?utm_source=note&utm_medium=referral&utm_campaign=YYYY-MM-DD`（campaignは記事の日付）。**「私たちについて」ブロック内のリンクも含めて全リンク対象。**
- **リンクは「裸のURLを単独の1行」に置く（2026-07-23制定・厳守）。**
  - NG: `AIside（https://aidollargame.com/aiside.html）：経営者の…` ← noteに貼ると全角カッコ以降の日本語までURLに飲み込まれてリンク切れ(404)になる。7/22記事で実際に発生。
  - OK: 説明文の行の**次の行**にURLだけを置く（`▼AIside：説明文` ＋ 改行 ＋ URL）。
- 「私たちについて」定型ブロックの正はこの形式（最新の `content/note/2026-07-23.md` の末尾ブロックをコピーして日付だけ変える）。
- 理由: GA4で「noteのどの記事が何人連れてきたか」を記事単位で計測するため。UTMなしだと note.com 全体のリファラーとしてしか見えず、効果検証ができない。
- 公開済み記事の差し替え状況（2026-07-23時点）: 7/22・7/15の2本は修正済み。7/8以前の8本は未了（editor.note.com/notes/{id}/edit/ で作業。手順の詳細はメモリのga4-weekly-reportに記録）。

## 機密の扱い

- 顧客名／契約／金額／補助金／案件の進捗は **Drive 台帳のみ**。
- このリポジトリ（`content/reminders.txt` を含む）には固有名詞の機密を書かない。

## 既知の制約

- 現状の Drive ツールは**新規作成のみ**で、既存ドキュメントの上書き編集ができない。
  そのため台帳の更新は「日付付きの新しい版を作る → 最新版を正本とする」方式。
  古い版の削除は手動（削除ツールが無いため）。在庫が増えたらユーザーに削除を促す。
